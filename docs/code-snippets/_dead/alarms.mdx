# Introduction

Alarms are used to monitor stack resources. With alarms you can easily:

- Specify the **metric** and the **resource** to be monitored, as well as what the **threshold** that causes alarm to
  fire.
- Setup email or slack notifications for when alarm is fired.
- Integrate alarm with your lambda function (trigger function on alarm)

# Usage

<PropertiesTable definitionName="Alarm" />

Alarm can be specified in `alarms` section of your config.

`embed:alarms/basic.stp.yml`

Alarm consists of 3 parts:

1. **Trigger** - Type of the trigger specifies what metric will be monitored. Depending on the type further properties
   of trigger can be specified
2. **Notifications** (OPTIONAL) - Specifies where to send notification when alarm is triggered
3. **Evaluation** (OPTIONAL) - Configures evaluation period (interval) for the monitored metric

# Integrating with function

You can integrate lambda function with alarm using cloudwatch-alarm trigger.

See also [lambda trigger events docs](/compute-resources/lambda-functions/#alarm-event).

`embed:lambda-functions/event-alarm.stp.yml`
