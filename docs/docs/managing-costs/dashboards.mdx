# Cost Dashboards

The Stacktape Console provides two cost views: a high-level overview with KPI cards and charts on the organization dashboard, and a detailed per-stack cost breakdown on the dedicated Costs per stack page. Both views use data from AWS Cost and Usage Reports, which are generated daily.

<Info>AWS bills infrastructure costs directly to your AWS account. Stacktape's subscription fee is separate and does not appear in these dashboards. See [Billing and subscription](/stacktape-console/billing-and-subscription) for subscription details.</Info>

This page covers **viewing** your AWS costs. For setting up spend alerts, see [Budgets](/managing-costs/budgets). For understanding which individual resources cost the most, see [Per-resource breakdown](/managing-costs/per-resource-breakdown). For reducing costs, see [Optimization tips](/managing-costs/optimization-tips).

## Organization overview

The organization overview dashboard includes cost-related KPI cards and charts alongside deployment and operational metrics. Cost data on the overview always reflects the **current calendar month** — the time range selector (7d, 14d, 30d, 90d, 6m, 1y) controls deployment statistics only, not cost views. The cost widgets give you a quick read on total spend, spend distribution, and budget status without navigating to the dedicated costs page.

### AWS Costs KPI

The AWS Costs KPI card displays your total spend for the current month with a month-over-month trend indicator. The trend compares current spend against the previous month's total and displays one of these labels:

| Trend label | Meaning |
|---|---|
| `+X%` or `-X%` | Percentage increase or decrease vs the previous month |
| `Flat` | Less than 0.1% change in either direction |
| `New spend` | Spend this month but no previous-month data to compare |
| `No spend` | Zero spend in both the current and previous month |

A sparkline below the card shows a recent monthly cost trend, giving you a quick visual of whether spend is rising or falling. The subtitle reads "This month" and includes the trend label for context.

### AWS Costs by Project

A horizontal bar chart titled "AWS Costs by Project" displays up to 10 items sorted by current-month spend. If more than 10 items have costs, the remainder is grouped under "Other." Hover over any bar to see the exact amount and name. This chart helps you identify which applications drive the largest share of your bill at a glance.

### AWS Costs by Resource

A separate horizontal bar chart titled "AWS Costs by Resource" breaks down current-month spend by AWS resource type. This view is useful for identifying which infrastructure category drives your bill, independent of which project uses it. If a particular resource type suddenly spikes, you know where to start investigating — even before checking which project is responsible.

### Budget Alerts

The Budget Alerts panel shows all configured [budget alerts](/managing-costs/budgets) with their name, scope (organization-wide or per stack), threshold amount, and the percentage levels that trigger notifications. Clicking a budget row navigates to the Console budgets page. When no budgets are configured, the panel displays "No budget alerts configured."

## Costs per stack

The dedicated Costs per stack page — accessible from the Console sidebar — provides a month-by-month table of AWS spend attributed to each deployed stack. You select a connected AWS account and a month, and the table shows every stack with its project name, stage, region, and total attributed cost for that period.

<ConsoleScreenshot src="/static/screenshots/console-costs-per-stack.png" alt="Costs per stack page showing a table with columns for project name, stage, region, and month-to-date costs, with AWS account and month filters at the top" caption="The Costs per stack page shows attributed AWS spend by project, stage, and region." />

### Filters

Two filters at the top of the page control the displayed data:

- **AWS account** — select from your [connected AWS accounts](/stacktape-console/connecting-your-aws-account). Switch between accounts to compare spend across different AWS environments.
- **Time range** — select a month. The selector includes each month so far in the current year plus all 12 months of the previous year — between 13 and 24 selectable months depending on when in the year you view it.

### Cost table

The table shows one row per stack with the following columns:

| Column | Description |
|---|---|
| Project name | The Stacktape project the stack belongs to |
| Stage | The stage name (e.g., `production`, `staging`) |
| Region | The AWS region the stack is deployed in |
| Month-to-date costs | Total attributed spend for the selected month |

The table sorts by cost descending by default and paginates at 10 rows per page. Clicking a row navigates to that project and stage's per-resource cost view.

<Warning>Reports are generated daily from AWS Cost and Usage Reports and may take up to a day to appear. If you deployed a new stack recently, its costs may not show up immediately.</Warning>

## Per-stage cost detail

Clicking a row in the Costs per stack table opens a per-resource cost breakdown for that specific stage. This view shows each resource within the stage — its name, type, and cost for the selected month — so you can pinpoint exactly which resource within a project drives spend.

The per-stage cost table has three columns:

| Column | Description |
|---|---|
| Resource name | The Stacktape resource name within the stage |
| Type | The AWS resource type |
| Costs | Cost for the selected month, shown to four decimal places |

A total is displayed at the top of the table. The time range selector uses the same month list as the Costs per stack page — each month of the current year plus all 12 months of the previous year. The table paginates at 15 rows per page and sorts by cost descending by default.

This view answers questions like "is my database or my container service the bigger cost driver in staging?" For guidance on acting on these findings, see [Optimization tips](/managing-costs/optimization-tips).

## FAQ

### How often is cost data updated?

Reports are generated daily from AWS Cost and Usage Reports and may take up to a day to appear. After a new deployment, there may be a delay before its costs show in the Console. Reloading the page fetches the latest available report data.

### Does Stacktape add any markup to AWS costs?

No. The costs shown in the dashboards are your actual AWS charges, billed directly to your AWS account. Stacktape's subscription fee is billed separately through the [billing page](/stacktape-console/billing-and-subscription) and does not appear in cost dashboards.

### Can I see costs for a specific stage?

Yes. The Costs per stack table lists each stage as a separate row. Click any row to open a per-resource cost breakdown for that stage, where each resource is listed with its individual cost.

### Why don't I see costs for a newly deployed stack?

AWS Cost and Usage Reports have a processing delay — reports are generated daily and may take up to a day to appear. If you deployed a stack recently, check back later. Additionally, some AWS services like Lambda only incur costs on invocation — a deployed but idle function may legitimately show zero cost.

### What is the difference between "AWS Costs by Project" and "AWS Costs by Resource"?

"AWS Costs by Project" groups current-month spend by project — useful for comparing application-level costs. "AWS Costs by Resource" groups spend by AWS resource type — useful for identifying which infrastructure category (compute, database, storage) drives your bill. Both charts appear on the organization overview.

### How is this different from the AWS Billing console?

The AWS Billing console shows raw, account-wide cost data without project-level attribution. The Stacktape Console automatically attributes costs to your specific projects and stages, so you can see per-stack spend without manually configuring AWS Cost Allocation Tags or building custom Cost Explorer reports. For deeper AWS-native analysis, you can use AWS Cost Explorer alongside the Stacktape dashboards.

### Can I set up alerts when costs exceed a threshold?

Yes. Stacktape supports budget alerts that notify you when spend reaches specified percentage thresholds. You can scope budgets to the entire organization or to individual stacks. See [Budgets](/managing-costs/budgets) for setup details.

### How far back can I view cost history?

The time range selector on both the Costs per stack page and the per-stage detail view includes each month so far in the current year plus all 12 months of the previous year. This gives you between 13 and 24 selectable months depending on when in the year you view it.

### What costs appear in the Costs per stack table?

The Costs per stack table displays attributed stack costs from the selected connected AWS account for the selected month. Each row represents a deployed stack with its total attributed spend. If no cost report is available for the selected account and time range, the table shows a message noting that reports are generated daily.

### Can I compare costs across multiple AWS accounts?

Yes. If you have multiple [connected AWS accounts](/stacktape-console/connecting-your-aws-account), use the AWS account dropdown on the Costs per stack page to switch between them. Each account's cost data is displayed independently — select a different account to compare spend across environments.
