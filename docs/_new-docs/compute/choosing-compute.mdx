# Choosing Compute Resources

Stacktape supports multiple compute options, from serverless functions to containers. Choose based on your workload characteristics.

## Comparison

| Resource            | Best For                          | Scaling         | Min Cost         | Cold Start |
| ------------------- | --------------------------------- | --------------- | ---------------- | ---------- |
| **Lambda Function** | APIs, event handlers, short tasks | Instant (1000s) | $0 (pay-per-use) | Yes        |
| **Edge Lambda**     | CDN customization, low latency    | Instant         | $0 (pay-per-use) | Yes        |
| **Web Service**     | APIs, websites, long-running      | Minutes (auto)  | ~$8/month        | No         |
| **Private Service** | Internal APIs, microservices      | Minutes (auto)  | ~$8/month        | No         |
| **Worker Service**  | Background processing             | Minutes (auto)  | ~$8/month        | No         |
| **Multi-Container** | Complex workloads, sidecars       | Minutes (auto)  | ~$8/month        | No         |
| **Batch Job**       | Data processing, ML training      | Minutes         | $0 (pay-per-use) | Yes        |
| **Next.js Web**     | Next.js applications              | Instant         | ~$5/month        | Minimal    |

### Lambda Functions

| Packaging Type               | Description                                      |
| ---------------------------- | ------------------------------------------------ |
| `stacktape-lambda-buildpack` | Zero-config bundling for JS/TS, Python, Java, Go |
| `custom-artifact`            | Your own pre-built deployment package            |

### Container Services

| Packaging Type              | Description                              |
| --------------------------- | ---------------------------------------- |
| `stacktape-image-buildpack` | Zero-config Docker image builder         |
| `external-buildpack`        | Cloud Native Buildpacks (Paketo, Heroku) |
| `nixpacks`                  | Nixpacks-based builds                    |
| `custom-dockerfile`         | Your own Dockerfile                      |
| `prebuilt-image`            | Existing Docker image                    |

See each resource's documentation for detailed packaging options.
