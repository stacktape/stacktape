# Configure Your Stack

A `stacktape.ts` file defines your entire AWS infrastructure — APIs, functions, databases, containers — using typed TypeScript classes. Stacktape reads this file, provisions every resource in your AWS account, and gives you full IDE autocompletion and type checking. This page walks you through creating one.

<Info>Already have a `stacktape.ts`? Skip to [using dev mode](/getting-started/use-the-dev-mode) or [deploying your first stage](/getting-started/deploy-your-first-stage).</Info>

## Install the package

Add `stacktape` to your project. The package includes all TypeScript resource classes, directive functions, and type definitions for your config.

```bash
npm install stacktape --save-dev
```

## Your first stacktape.ts

Create a `stacktape.ts` file in your project root. This example deploys a [Lambda function](/resources/compute/lambda-function) behind an [HTTP API Gateway](/resources/networking/http-api-gateway) — a common pattern for REST APIs:

<CodeBlock
  intellisense
  tabs={[
    {
      label: 'TypeScript',
      lang: 'typescript',
      code: `import {
  defineConfig,
  HttpApiGateway,
  LambdaFunction,
  StacktapeLambdaBuildpackPackaging,
  HttpApiIntegration
} from 'stacktape';

// [!code focus-start]
export default defineConfig(() => {
  const api = new HttpApiGateway({});

  const myFunction = new LambdaFunction({
    packaging: new StacktapeLambdaBuildpackPackaging({
      entryfilePath: './src/handler.ts'
    }),
    memory: 512,
    timeout: 30,
    events: [
      new HttpApiIntegration({
        httpApiGatewayName: api.resourceName,
        method: 'GET',
        path: '/hello'
      })
    ]
  });

  return {
    resources: { api, myFunction }
  };
});
// [!code focus-end]
`
    }
  ]}
/>

Here's what each piece does:

- **`defineConfig`** wraps your config in a function that Stacktape calls at deploy time. It provides deployment context (stage, region, project name) as parameters.
- **Resource classes** like `LambdaFunction` and `HttpApiGateway` map directly to AWS resources. Each class accepts a typed configuration object with autocompletion.
- **`StacktapeLambdaBuildpackPackaging`** tells Stacktape how to build your code. Point it at a TypeScript or JavaScript entrypoint and Stacktape bundles it automatically — no build config needed. See [function packaging](/packaging/function/stacktape-buildpack) for details.
- **`HttpApiIntegration`** connects HTTP requests to your function through the API Gateway. The `api.resourceName` reference links the two resources.
- **The `resources` return** registers everything in your stack. The object keys (`api`, `myFunction`) become the resource names — they determine CloudFormation logical IDs and the prefix for auto-injected environment variables (e.g., `STP_MY_FUNCTION_ARN`).

The handler file at `./src/handler.ts` is standard application code:

```typescript
// src/handler.ts
export default async () => {
  return {
    statusCode: 200,
    body: JSON.stringify({ message: 'Hello from Stacktape!' })
  };
};
```

## Stage-based configuration

The `defineConfig` callback receives a context object with `stage`, `region`, `projectName`, `command`, and `cliArgs`. Use these to vary infrastructure per [stage](/configuration/stages-and-environments) — smaller instances in development, larger ones in production:

<CodeBlock
  intellisense
  tabs={[
    {
      label: 'TypeScript',
      lang: 'typescript',
      code: `import { defineConfig, RelationalDatabase, RdsEnginePostgres, $Secret } from 'stacktape';

// [!code focus-start]
export default defineConfig(({ stage }) => {
  const database = new RelationalDatabase({
    engine: new RdsEnginePostgres({
      version: '16',
      primaryInstance: {
        instanceSize: stage === 'production' ? 'db.t4g.medium' : 'db.t4g.micro'
      }
    }),
    credentials: {
      masterUserPassword: $Secret('db-password')
    }
  });

  return {
    resources: { database }
  };
});
// [!code focus-end]`
    }
  ]}
/>

The `$Secret` directive references a secret stored in the [Stacktape Console](/configuration/secrets) — it resolves at deploy time so credentials never appear in your config file. Because your config is TypeScript, you can use conditionals, loops, shared modules, and any other language feature to keep things DRY. See [variables and reuse](/configuration/variables-and-reuse) for common patterns.

| Context property | Type       | Description                                            |
| ---------------- | ---------- | ------------------------------------------------------ |
| `stage`          | `string`   | Current stage name (e.g., `dev`, `production`)         |
| `region`         | `string`   | AWS region (e.g., `us-east-1`, `eu-west-1`)           |
| `projectName`    | `string`   | Project name                                           |
| `command`        | `string`   | CLI command being run (`deploy`, `delete`, `dev`, etc.)|
| `cliArgs`        | `object`   | Additional CLI arguments                               |

## Quick-start with stacktape init

If you'd rather start from a working template, [`stacktape init`](/cli/init) analyzes your project directory and generates a `stacktape.ts` tailored to your codebase:

```bash
npx stacktape@latest init
```

The wizard auto-detects your tech stack — framework, language, dependencies — and generates a config with the right resources, packaging, and estimated costs. You can also bootstrap from a pre-built starter project using `--starterId`:

```bash
npx stacktape@latest init --starterId nextjs-serverless
```

Starters include REST APIs with databases, Next.js apps, container services, background workers, and more. The generated project comes with a working `stacktape.ts`, application code, and dependencies — ready to deploy immediately.

## Console alternatives

The [Stacktape Console](/stacktape-console/console-overview) offers two additional paths for creating a config:

**AI config generation** — Point the Console at any Git repository and branch. Stacktape analyzes the codebase and generates a `stacktape.ts` tailored to your project's framework, language, and dependencies. Works with both public repositories (paste a URL) and private repositories (connect your Git provider). This is the fastest path when you have an existing codebase and want Stacktape to figure out the infrastructure. See [AI config generation](/using-with-ai/config-generation-from-repository) for details.

**Visual config editor** — A browser-based Monaco editor with full IntelliSense, inline validation, and live price estimation. Write or edit your config directly in the Console. The editor shows estimated monthly AWS costs as you add resources, which helps you understand cost impact before deploying. See [visual config editor](/stacktape-console/visual-config-editor) for the walkthrough.

Both are useful for exploration and getting started quickly. For production workflows, keep your `stacktape.ts` in version control alongside your application code — it's the single source of truth for your infrastructure.

## What you can configure

Beyond `resources`, a Stacktape config supports several top-level sections:

- **[scripts](/configuration/hooks-and-scripts)** — shell commands or code you can run manually with `stacktape script:run` or attach to lifecycle hooks. Common uses: database migrations, seed scripts, build steps.
- **[hooks](/configuration/hooks-and-scripts)** — run scripts automatically before or after `deploy`, `delete`, or `dev` commands. Example: run Prisma migrations after every deploy.
- **[variables](/configuration/variables-and-reuse)** — reusable values you can reference anywhere with `$Var().variableName`.
- **[directives](/configuration/directives)** — built-in functions like `$Secret()`, `$ResourceParam()`, and `$File()` that resolve dynamic values at deploy time.
- **[cloudformationResources](/configuration/overrides-and-escape-hatches)** — an escape hatch to add raw CloudFormation resources alongside Stacktape-managed ones.

You don't need any of these to get started — a config with just `resources` deploys a working stack. Add the other sections as your project grows.
