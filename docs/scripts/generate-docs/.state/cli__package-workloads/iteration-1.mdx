# package-workloads

The `package-workloads` command builds and packages all compute resources defined in your Stacktape configuration — Lambda functions, container workloads, batch jobs — without deploying them. Use it to inspect the resulting artifacts (zips, container images) before running a full [`deploy`](/cli/deploy).

## Usage

```bash
stacktape package-workloads --stage dev --region eu-west-1
```

The command reads your Stacktape configuration, runs the same packaging pipeline that [`deploy`](/cli/deploy) uses, and writes the artifacts to the local build output directory. No CloudFormation stack is created or updated, and nothing is uploaded to AWS.

## When to use

Run `package-workloads` when you need to verify what Stacktape will produce before committing to a deployment. Common scenarios:

- **Checking bundle size.** Confirm that a Lambda zip stays under the 50 MB direct-upload limit or that tree-shaking excluded the right dependencies.
- **Debugging packaging issues.** If a deploy fails during the build phase, isolate the problem by packaging first — without waiting for CloudFormation.
- **CI artifact inspection.** In a CI pipeline, package first, run checks or scans on the artifacts, then deploy in a separate step.

<Info>

The command always runs without build cache (`commandCanUseCache: false`), so every invocation produces a fresh build regardless of prior runs.

</Info>

## Important flags

The two required flags are `--stage` and `--region`. Both are needed because Stacktape resolves stage-specific configuration (secrets, directives, resource names) during packaging.

Point to a non-default config file with `--configPath`.

```bash
stacktape package-workloads --stage prod --region us-east-1 --configPath ./infra/stacktape.ts
```

Use `--profile` to authenticate with a specific AWS profile.

```bash
stacktape package-workloads --stage dev --region eu-west-1 --profile my-aws-profile
```

Use `--logLevel debug` to see detailed build output — useful when diagnosing bundler or Dockerfile issues.

```bash
stacktape package-workloads --stage dev --region eu-west-1 --logLevel debug
```

## Examples

Package all workloads for a staging stage in `eu-west-1`.

```bash
stacktape package-workloads --stage staging --region eu-west-1
```

Package workloads using a specific AWS account connected in the Stacktape Console.

```bash
stacktape package-workloads --stage prod --region us-east-1 --awsAccount production-account
```

Package with machine-readable JSONL output for CI integration.

```bash
stacktape package-workloads --stage dev --region eu-west-1 --outputFormat jsonl
```
