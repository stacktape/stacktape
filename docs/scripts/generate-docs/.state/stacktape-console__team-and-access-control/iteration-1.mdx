# Team and Access Control

The Stacktape Console lets you invite team members to your organization, assign roles that control what each person can do, and scope access to specific projects. Every member gets a role — Owner, Admin, Developer, or Viewer — and the permission system enforces what they can view, deploy, and manage across your infrastructure.

## What this part of the Console does

The Users page displays all members of your organization in a table showing each member's name, email, role, creation date, account status, and MFA status. From this page you can invite new members, change roles, revoke API keys, and remove members — each action gated by your own role's permissions.

## Roles

Stacktape uses four active roles with a clear hierarchy. Each role includes all permissions of the roles below it, plus additional capabilities.

| Role | Scope | Description |
|------|-------|-------------|
| **Owner** | Organization-wide | Full control. Can delete the organization, manage billing, and transfer ownership. |
| **Admin** | Organization-wide | Everything except organization deletion. Can manage members, AWS accounts, guardrails, and integrations. |
| **Developer** | Project-scopable | Can deploy, delete non-production stages, manage secrets, and run scripts. |
| **Viewer** | Project-scopable | Read-only. Can view projects, deployments, logs, issues, and costs. |

<Info>Owner and Admin roles always have access to all projects. Developer and Viewer roles can be scoped to specific projects, restricting what the member can see and act on.</Info>

### Choosing the right role

Assign **Owner** only to founders or team leads who need billing and organization-deletion access — most teams have one or two Owners. Use **Admin** for senior engineers or DevOps leads who manage AWS accounts, invite members, and configure guardrails but should not be able to delete the organization itself. **Developer** fits most engineers who deploy code and manage secrets for their projects. **Viewer** works for stakeholders, QA, or on-call responders who need to read logs and metrics without changing infrastructure.

## Walkthrough: inviting a team member

1. In the Stacktape Console, open the **Users** page for your organization.
2. Click **Invite new member** in the top-right corner.
3. Fill in the member's full name, email address, and select a role.
4. If you selected Developer or Viewer, an additional **Project access** field appears. Choose which projects this member can access, or leave it empty to grant access to all current projects.
5. Click **Invite**. The new member receives an email invitation and appears in the table with status "Invited" until they confirm.

<Warning>You cannot invite members to a personal (default) organization. Create a named organization first, then invite your team there.</Warning>

### Project-scoped access

Developer and Viewer roles support project-level scoping. When you assign a member to specific projects, they can only see and interact with those projects — everything else in the organization is hidden from them. Owner and Admin roles always have organization-wide access and cannot be project-scoped.

## Common tasks

### Changing a member's role

Click the edit icon next to any member in the Users table. Select the new role from the dropdown. If the new role is project-scopable (Developer or Viewer), you can also reassign their project access. Admins cannot promote a member to Owner — only existing Owners can assign the Owner role.

### Removing a member

Click the delete icon next to the member you want to remove. A confirmation dialog shows the member's name and current role. Confirming removal immediately revokes their access to the organization. Their API keys remain active until explicitly revoked.

### Revoking a member's API keys

Click the key-revoke icon next to a member to invalidate all their active [API keys](/stacktape-console/api-keys) in this organization. This is useful when offboarding a team member or responding to a compromised credential. The revocation takes effect immediately — any CI/CD pipelines or CLI sessions using those keys will fail on the next request.

<Tip>When offboarding someone, revoke their API keys before or at the same time as removing them. Removing a member alone does not automatically revoke their API keys.</Tip>

### Checking MFA status

The Users table shows each member's MFA (multi-factor authentication) status. Members using an external identity provider show "External" in the MFA column. For email/password accounts, the column shows "Enabled" or "Disabled". You cannot force MFA from this page — members enable it in their own account settings.

## Member limits

The number of members you can invite depends on your subscription plan:

| Plan | Member limit |
|------|-------------|
| Free (personal org) | 1 (no invitations) |
| Flexible | 25 per organization |
| Organization (Enterprise) | Unlimited |

If you've reached your plan's member limit, the **Invite new member** button is disabled. Upgrade your plan on the [Billing and Subscription](/stacktape-console/billing-and-subscription) page to increase the limit.

## Permissions reference

Each role grants a specific set of permissions. The table below shows the key permission areas and which roles have access.

| Permission area | Owner | Admin | Developer | Viewer |
|----------------|-------|-------|-----------|--------|
| Delete organization | Yes | No | No | No |
| Manage billing | Yes | Yes | No | No |
| Manage AWS accounts | Yes | Yes | No | No |
| Invite/remove members | Yes | Yes | No | No |
| Update member roles | Yes | Yes | No | No |
| Manage guardrails | Yes | Yes | No | No |
| Deploy stacks | Yes | Yes | Yes | No |
| Delete production stages | Yes | Yes | No | No |
| Delete non-production stages | Yes | Yes | Yes | No |
| Manage secrets | Yes | Yes | Yes | No |
| Manage GitOps configs | Yes | Yes | Yes | No |
| View projects and logs | Yes | Yes | Yes | Yes |
| View issues | Yes | Yes | Yes | Yes |
| Manage own API keys | Yes | Yes | Yes | No |
| Revoke all user API keys | Yes | Yes | No | No |

<Info>Developers can delete non-production stages but not production stages. This lets engineering teams iterate freely on dev/staging environments while protecting production from accidental deletion.</Info>

## Troubleshooting

### "You do not have permission to invite members"

Your role does not include the `members:invite` permission. Only Owners and Admins can invite new members. Ask an Owner or Admin in your organization to either invite the member for you or upgrade your role.

### Invite button disabled on a non-personal organization

You've reached the maximum member count for your subscription plan. Check [Billing and Subscription](/stacktape-console/billing-and-subscription) to see your current plan and upgrade options.

### Removed member still has API access

Removing a member from the organization does not automatically revoke their API keys. Use the key-revoke action on the Users page (or ask an Admin/Owner to do so) to invalidate all their active keys.

### Member shows "Invited" status indefinitely

The invited member hasn't confirmed their account yet. They need to check their email for the invitation and complete the sign-up flow. If the invitation email was lost, remove and re-invite them.

## FAQ

### How many team members can I invite?

Member limits depend on your subscription plan. The Free plan on a personal organization allows only 1 member (yourself). The Flexible plan supports up to 25 members per organization. The Organization (Enterprise) plan has no member limit. You can check and upgrade your plan on the [Billing and Subscription](/stacktape-console/billing-and-subscription) page.

### Can I restrict a developer to specific projects?

Yes. The Developer and Viewer roles support project-level scoping. When inviting or editing a member with one of these roles, you can select which projects they have access to. Members with project-scoped access cannot see or interact with other projects in the organization.

### What happens when I remove a member?

Removing a member immediately revokes their access to the organization in the Console. However, their API keys remain valid until explicitly revoked. Always revoke API keys when offboarding someone to prevent continued CLI or CI/CD access.

### Can an Admin promote someone to Owner?

No. Only existing Owners can assign the Owner role to another member. This prevents privilege escalation — Admins have nearly full control but cannot grant themselves or others the ability to delete the organization or manage ownership transfers.

### Is there an audit log of member actions?

Yes. The Activity page in the Stacktape Console shows a log of CLI operations (deploys, deletes, scripts, and other recorded commands) across your organization, attributed to the member who ran them. This helps track who deployed what and when.

### Does Stacktape support SSO (Single Sign-On)?

SSO is available on the Organization (Enterprise) plan. Contact sales for setup. Members authenticating via an external identity provider show "External provider" as their account status and "External" for MFA on the Users page.

### What's the difference between revoking API keys and removing a member?

Revoking API keys invalidates all active keys a member has created — their CLI sessions and CI/CD pipelines stop working immediately. Removing a member revokes Console access but leaves API keys intact until separately revoked. For complete offboarding, do both.

### Can I enforce MFA for all organization members?

The Users page shows MFA status per member but does not currently offer an organization-wide MFA enforcement toggle. Each member enables MFA in their own account settings. If you need mandatory MFA, the Organization (Enterprise) plan with SSO lets you enforce it through your identity provider.
