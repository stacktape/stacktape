# upgrade

The `stacktape upgrade` command upgrades your Stacktape CLI installation to the latest available version. It detects how Stacktape was installed (native script, global package, or local dependency) and runs the appropriate upgrade mechanism automatically. No API key or AWS credentials are required.

## Usage

```bash
stacktape upgrade
```

To install a specific version instead of the latest:

```bash
stacktape upgrade --newVersion 3.5.0
```

## How it works

The command compares your current version against the latest published release using semver. If you're already on the latest version, it exits with a success message and takes no action.

When an upgrade is available, Stacktape detects your installation type and acts accordingly:

- **Native installation** (`~/.stacktape/bin`): runs the install script directly to replace the binary.
- **Global package** (npm, pnpm, or bun): runs the corresponding global update command (e.g. `npm update -g stacktape`).
- **Local project dependency**: prints the correct update command for you to run manually, since modifying project dependencies requires your confirmation.
- **Unknown installation method**: prints the native install script as a fallback.

On Windows, upgrade commands execute through PowerShell. On macOS and Linux, they execute through `sh`.

## Examples

Upgrade to the latest version:

```bash
stacktape upgrade
```

Pin a specific version (useful when testing compatibility or rolling back a CLI update):

```bash
stacktape upgrade --newVersion 3.4.2
```

Short alias form:

```bash
stacktape upgrade --nv 3.4.2
```

<Info>
If Stacktape is installed as a local project dependency (in `node_modules`), the command prints the appropriate update command instead of running it. Run the printed command yourself to upgrade.
</Info>
