# debug:metrics

Fetch CloudWatch metrics for any deployed resource directly from the CLI. Use `debug:metrics` to check invocation counts, error rates, CPU usage, latency, and other key indicators without opening the AWS Console or setting up a dashboard.

## Usage

```bash
stacktape debug:metrics --stage prod --region eu-west-1 --resourceName myApi --metric Invocations
```

The command queries CloudWatch for the specified metric and renders a chart in your terminal (or returns JSON in agent mode). You must specify both the resource and the exact metric name.

## Supported resource types and metrics

The `debug:metrics` command supports four resource types. Each type exposes a specific set of CloudWatch metrics you can query.

| Resource type | Metrics |
|---|---|
| Lambda function | `Invocations`, `Errors`, `Duration`, `Throttles`, `ConcurrentExecutions` |
| Multi-container workload (ECS) | `CPUUtilization`, `MemoryUtilization` |
| Relational database (RDS) | `CPUUtilization`, `DatabaseConnections`, `FreeStorageSpace`, `ReadLatency`, `WriteLatency` |
| HTTP API Gateway | `Count`, `4XXError`, `5XXError`, `Latency`, `IntegrationLatency` |

If you pass a metric name that doesn't match the resource type, the command returns an error listing the available metrics for that resource.

## Important flags

**`--metric`** (required) — The CloudWatch metric name. Must be one of the supported metrics for the resource type (see table above).

**`--startTime`** — How far back to query. Accepts relative durations like `1h`, `30m`, `1d`, `60s`, or an absolute date string parseable by JavaScript's `Date` constructor. Defaults to 1 hour ago.

**`--period`** — Aggregation period in seconds. Each datapoint represents one period. Defaults to `300` (5 minutes). Use smaller values like `60` for finer granularity on short time ranges.

**`--stat`** — The statistic to compute for each period. Accepts `Sum`, `Average`, `Maximum`, `Minimum`, or percentiles like `p99`. Defaults to `Average`.

## Examples

Fetch Lambda error count over the last hour with 1-minute granularity:

```bash
stacktape debug:metrics --stage prod --region eu-west-1 --resourceName myApi --metric Errors --period 60 --stat Sum
```

Check database CPU over the last 24 hours:

```bash
stacktape debug:metrics --stage prod --region eu-west-1 --resourceName mainDatabase --metric CPUUtilization --startTime 1d
```

Get p99 latency for an API Gateway over the last 6 hours:

```bash
stacktape debug:metrics --stage prod --region eu-west-1 --resourceName gateway --metric Latency --startTime 6h --stat p99
```

Get JSON output for programmatic use (agent mode):

```bash
stacktape debug:metrics --stage prod --region eu-west-1 --resourceName myApi --metric Invocations --agent
```

## Agent mode output

When `--agent` is set, the command outputs a JSON object instead of a terminal chart. The JSON contains the resource name, metric, period, stat, and an array of `{ timestamp, value }` datapoints sorted chronologically. This is useful for feeding metrics into scripts, dashboards, or AI coding agents.
