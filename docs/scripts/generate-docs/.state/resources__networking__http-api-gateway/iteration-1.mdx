# HTTP API Gateway

An HTTP API Gateway is a serverless entry point for routing HTTP requests to Lambda functions or container workloads. Built on AWS API Gateway V2, it handles TLS termination, request throttling, CORS, and access logging with no infrastructure to manage. Define routes declaratively and pay only for requests processed.

**Pricing**: ~$1/million requests. No base cost — you pay nothing when there's no traffic.

## When to use

**Use an HTTP API Gateway when:**

- You're building an API from [Lambda functions](/resources/compute/lambda-function) and want a shared entry point with path-based routing
- You need pay-per-request pricing that scales to zero when idle
- You want managed CORS, TLS, and request throttling without writing middleware
- Each request completes within 29 seconds (the API Gateway integration timeout)

**Use something else when:**

| Situation | Better option |
|-----------|---------------|
| WebSocket or long-lived connections | [Application Load Balancer](/resources/networking/application-load-balancer) |
| Always-on container with public URL | [Web service](/resources/compute/web-service) — creates its own gateway by default |
| Non-HTTP protocols (TCP, TLS, MQTT) | [Network Load Balancer](/resources/networking/network-load-balancer) |
| Request processing exceeds 29 seconds | [Application Load Balancer](/resources/networking/application-load-balancer) or async pattern |
| Web application firewall (WAF) at API level | [Application Load Balancer](/resources/networking/application-load-balancer) with `useFirewall` |

<Info>A [web service](/resources/compute/web-service) creates an HTTP API Gateway automatically when using the default load balancing. If you have a single containerized app that handles all routing internally, use a web service instead of defining a gateway manually.</Info>

## Basic example

This creates an HTTP API Gateway with a Lambda function handling all requests:

<CodeBlock
  intellisense
  tabs={[
    {
      label: 'stacktape.ts',
      lang: 'typescript',
      code: `import { defineConfig, HttpApiGateway, LambdaFunction, StacktapeLambdaBuildpackPackaging, HttpApiIntegration } from 'stacktape';

// [!code focus-start]
export default defineConfig(() => {
  const gateway = new HttpApiGateway({});

  const api = new LambdaFunction({
    packaging: new StacktapeLambdaBuildpackPackaging({
      entryfilePath: './src/handler.ts'
    }),
    events: [
      new HttpApiIntegration({
        httpApiGatewayName: 'gateway',
        method: '*',
        path: '/{proxy+}'
      })
    ]
  });

  return {
    resources: { gateway, api }
  };
});
// [!code focus-end]`
    }
  ]}
/>

```typescript
// src/handler.ts
import { Hono } from 'hono';
import { handle } from 'hono/aws-lambda';

const app = new Hono();
app.get('/', (c) => c.json({ status: 'ok' }));
app.get('/users', (c) => c.json({ users: [] }));

export const handler = handle(app);
```

After deploying with `stacktape deploy`, you receive a public HTTPS URL for your API.

The `method: '*'` and `path: '/{proxy+}'` pattern routes all incoming requests to a single Lambda function, letting your framework handle routing internally. This is the most common pattern for APIs with many routes.

## Routing

Each Lambda function declares which HTTP method and path combinations it handles through `HttpApiIntegration` events. The gateway routes matching requests to the corresponding function.

Three path formats are supported:

- **Exact path**: `/users` — matches only `/users`
- **Path parameter**: `/users/{id}` — matches `/users/123`, accessible in `event.pathParameters.id`
- **Greedy path**: `/orders/{proxy+}` — matches any path starting with `/orders/`

Routes are matched by specificity — exact paths take priority over wildcard paths.

<CodeBlock
  intellisense
  tabs={[
    {
      label: 'stacktape.ts',
      lang: 'typescript',
      code: `import { defineConfig, HttpApiGateway, LambdaFunction, StacktapeLambdaBuildpackPackaging, HttpApiIntegration } from 'stacktape';

export default defineConfig(() => {
  const gateway = new HttpApiGateway({});

  // [!code focus-start]
  const usersApi = new LambdaFunction({
    packaging: new StacktapeLambdaBuildpackPackaging({
      entryfilePath: './src/users.ts'
    }),
    events: [
      new HttpApiIntegration({
        httpApiGatewayName: 'gateway',
        method: 'GET',
        path: '/users'
      }),
      new HttpApiIntegration({
        httpApiGatewayName: 'gateway',
        method: 'POST',
        path: '/users'
      }),
      new HttpApiIntegration({
        httpApiGatewayName: 'gateway',
        method: 'GET',
        path: '/users/{id}'
      })
    ]
  });

  const ordersApi = new LambdaFunction({
    packaging: new StacktapeLambdaBuildpackPackaging({
      entryfilePath: './src/orders.ts'
    }),
    events: [
      new HttpApiIntegration({
        httpApiGatewayName: 'gateway',
        method: '*',
        path: '/orders/{proxy+}'
      })
    ]
  });
  // [!code focus-end]

  return {
    resources: { gateway, usersApi, ordersApi }
  };
});`
    }
  ]}
/>

Each method + path combination must be unique across all integrations on the same gateway. Supported methods: `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `HEAD`, `OPTIONS`, or `*` (any method).

Each integration can specify an `authorizer` to protect the route — unauthorized requests receive a `401` response.

<Warning>Lambda functions behind an HTTP API Gateway have an effective timeout cap of 29 seconds — the API Gateway integration timeout. Even if your Lambda timeout is higher, the gateway returns a 504 after 29 seconds. For longer processing, use an [Application Load Balancer](/resources/networking/application-load-balancer) or an async pattern.</Warning>

<Info>Container workloads (multi-container-workload) can also route through an HTTP API Gateway, with an additional `containerPort` property on each integration. All integrations on a container workload must target the same container port, and only payload format `1.0` is supported.</Info>

## CORS

The `cors` property configures Cross-Origin Resource Sharing at the gateway level, overriding any CORS headers set by your application code.

<CodeBlock
  intellisense
  tabs={[
    {
      label: 'stacktape.ts',
      lang: 'typescript',
      code: `import { defineConfig, HttpApiGateway } from 'stacktape';

export default defineConfig(() => {
  const gateway = new HttpApiGateway({
    // [!code focus-start]
    cors: {
      enabled: true,
      allowedOrigins: ['https://myapp.com'],
      allowCredentials: true
    }
    // [!code focus-end]
  });

  return {
    resources: { gateway }
  };
});`
    }
  ]}
/>

When `cors.enabled` is `true` with no other options, the gateway uses permissive defaults: `*` origins, common headers (`Content-Type`, `Authorization`, `X-Api-Key`, and others), and methods auto-detected from your integrations.

## Custom domains

Attach your own domain (e.g., `api.example.com`) to the gateway. Stacktape creates the DNS record and provisions a free TLS certificate automatically.

Your domain must have a Route53 hosted zone in your AWS account, with your registrar's nameservers pointing to it.

<CodeBlock
  intellisense
  tabs={[
    {
      label: 'stacktape.ts',
      lang: 'typescript',
      code: `import { defineConfig, HttpApiGateway } from 'stacktape';

export default defineConfig(() => {
  const gateway = new HttpApiGateway({
    // [!code focus-start]
    customDomains: [{ domainName: 'api.example.com' }]
    // [!code focus-end]
  });

  return {
    resources: { gateway }
  };
});`
    }
  ]}
/>

To use your own TLS certificate (e.g., EV/OV certs), set `customCertificateArn` to an ACM certificate ARN. To manage DNS records yourself (e.g., through Cloudflare), set `disableDnsRecordCreation: true`.

## CDN

Add a CDN (CloudFront) in front of the gateway for edge caching and lower latency worldwide. The CDN has no monthly base cost — you pay per request and per GB transferred.

## Access logging

Access logs record request metadata — request ID, client IP, HTTP method, status code, protocol, and response length. Logs are viewable with `stacktape logs` or in the Stacktape Console.

<CodeBlock
  intellisense
  tabs={[
    {
      label: 'stacktape.ts',
      lang: 'typescript',
      code: `import { defineConfig, HttpApiGateway } from 'stacktape';

export default defineConfig(() => {
  const gateway = new HttpApiGateway({
    // [!code focus-start]
    logging: {
      format: 'JSON',
      retentionDays: 90
    }
    // [!code focus-end]
  });

  return {
    resources: { gateway }
  };
});`
    }
  ]}
/>

Defaults: format `JSON`, retention `30` days, logging enabled. Set `disabled: true` to turn off access logging. Supported formats: `CLF`, `JSON`, `XML`, `CSV`.

## Payload format

The `payloadFormat` property controls the shape of the event object your Lambda function receives:

- **`1.0`** (default) — includes `multiValueHeaders` and `multiValueQueryStringParameters`. Compatible with API Gateway v1 proxy format.
- **`2.0`** — simpler structure with flat headers and query string. Recommended for new projects.

Set `payloadFormat` on the gateway to apply it globally, or override it per-integration on individual Lambda functions. All integrations on a single Lambda function must use the same payload format.

## Alarms

Two alarm types are available for HTTP API Gateways:

- **Error rate** (`http-api-gateway-error-rate`) — fires when 4xx/5xx error rate exceeds a percentage threshold
- **Latency** (`http-api-gateway-latency`) — fires when request-to-response time exceeds a millisecond threshold

Configure alarms in the `alarms` property, or use global alarms from the Stacktape Console. Use `disabledGlobalAlarms` to exclude specific global alarms from this gateway.
