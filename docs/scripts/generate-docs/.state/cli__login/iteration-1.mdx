# login

The `stacktape login` command configures your Stacktape API key on the current system. After logging in, all subsequent Stacktape operations are associated with the user and organization linked to that API key. You can provide the key directly via `--apiKey` or enter it through an interactive authentication flow.

## Usage

```bash
stacktape login
```

In interactive mode (the default), the command launches an auth flow that supports Google sign-in, email signup, email login, or manual API key entry. After successful authentication, Stacktape displays your user name and organization.

To skip the interactive flow, pass the API key directly.

```bash
stacktape login --apiKey sk-your-api-key
```

This is useful in CI/CD environments or when scripting. You can obtain your API key from the [Stacktape Console's API keys page](/stacktape-console/api-keys).

## What happens during login

Stacktape performs these steps when you run `login`:

1. **Collects your API key** — either from the `--apiKey` flag or through the interactive auth flow.
2. **Verifies the key** — calls the Stacktape API to confirm the key is valid and retrieves your user and organization data.
3. **Saves the key locally** — stores the API key on your system so subsequent commands use it automatically.
4. **Displays your identity** — prints your user name and the organization the key belongs to.

If the API key is invalid, Stacktape clears it from local storage and reports an authentication error.

<Info>
The `login` command itself does not require an existing API key — it is one of the commands that can run without prior authentication.
</Info>

## Non-interactive login

For CI/CD pipelines, automated scripts, or AI agent workflows, use the `--apiKey` flag combined with `--agent` for machine-readable output.

```bash
stacktape login --apiKey sk-your-api-key --agent
```

The `--agent` flag outputs strict JSONL and disables the interactive terminal UI, making it safe for non-TTY environments.

## Examples

### Interactive login

Run `login` with no flags to start the interactive authentication flow. You will be guided through sign-in options including Google, email, or manual API key entry.

```bash
stacktape login
```

### Login with an API key in CI

Pass the API key directly, typically from a CI secret or environment variable.

```bash
stacktape login --apiKey "$STACKTAPE_API_KEY"
```

### Verify your identity after login

After logging in, confirm which user and organization are active.

```bash
stacktape info:whoami
```

### Switch organizations

If your API key is scoped to a specific organization, log in with a different key to switch. Each API key is tied to one organization. You can create a new organization with [`org:create`](/cli/org-create).

```bash
stacktape login --apiKey sk-different-org-key
```

## Related commands

- [`logout`](/cli/logout) — removes the stored API key from the current system.
- [`info:whoami`](/cli/info-whoami) — displays the current user, organization, and connected AWS accounts.
- [`org:create`](/cli/org-create) — creates a new organization and returns an API key scoped to it.
- [`org:list`](/cli/org-list) — lists all organizations accessible with your current user.

## FAQ

### Where do I find my Stacktape API key?

API keys are managed in the [Stacktape Console](https://console.stacktape.com/api-keys). Navigate to the API keys section to create or copy an existing key. Each key is scoped to one organization. See [API keys](/stacktape-console/api-keys) for details.

### Does `login` connect my AWS account?

No. The `login` command only authenticates you with Stacktape. Connecting an AWS account to your organization is a separate step done through the [Stacktape Console](/stacktape-console/connecting-your-aws-account). You need both a valid Stacktape login and a connected AWS account before you can deploy.

### Can I use `login` in a CI/CD pipeline?

Yes. Pass the API key via `--apiKey` so no interactive prompt is needed. Combine with `--agent` for machine-readable JSONL output. Store the key as a CI secret and reference it as an environment variable.

### What happens if I log in with an invalid API key?

Stacktape attempts to verify the key against the API. If verification fails, the command clears the invalid key from local storage and reports an authentication error. No stale or invalid key is persisted.

### How do I switch between organizations?

Each API key is scoped to one organization. To switch, generate a new API key in the target organization's Console and run `stacktape login --apiKey <new-key>`. Use [`org:list`](/cli/org-list) to see which organizations you have access to.

### Does login expire?

The API key persists on your system until you run [`logout`](/cli/logout) or overwrite it with a new `login`. The key itself does not expire automatically, but it can be revoked from the [Stacktape Console](/stacktape-console/api-keys).
