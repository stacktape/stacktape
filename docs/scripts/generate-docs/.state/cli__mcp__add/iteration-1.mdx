# mcp:add

The `mcp:add` command installs the Stacktape MCP server configuration into coding-agent clients on your machine. It detects supported clients, adds or updates a `stacktape` MCP server entry pointing to `stacktape mcp`, and creates timestamped backups before modifying any existing configuration files.

## Usage

```bash
stacktape mcp:add
```

In interactive mode, the command presents a multi-select prompt listing detected clients and asks for confirmation before writing. In agent mode or with `--autoConfirmOperation`, it skips prompts and installs to all detected clients (or all supported clients if none are detected).

## Supported clients

The command detects and configures the following coding-agent clients:

| Client | Config format | Config locations |
|--------|--------------|-----------------|
| Claude Code | JSON | `.mcp.json` (project), `~/.claude.json` (user) |
| OpenAI Codex | TOML | `.codex/config.toml` (project), `~/.codex/config.toml` (user) |
| Cursor | JSON | `.cursor/mcp.json` (project), `~/.cursor/mcp.json` (user) |
| VS Code / Copilot | JSON | `.vscode/mcp.json` (project), platform-specific user path |
| OpenCode | JSON | `opencode.jsonc` (project), platform-specific user path |
| Windsurf | JSON | `.codeium/windsurf/mcp_config.json` (project), user-level equivalent |

For each client, the command checks the project-local path first. If an existing config file is found there, it uses that path. Otherwise it falls back to the user-level path.

## Behavior details

**Local vs global command resolution** — When installing into a project-local config file (e.g., `.mcp.json` in the current directory), the command checks whether `stacktape` is listed as a dependency in `package.json`. If so, it writes `node_modules/.bin/stacktape` as the command path so the MCP server uses the project-local Stacktape version. For user-level config files, it always uses the global `stacktape` command.

**Backups** — Before modifying an existing file, the command copies it to a timestamped backup (e.g., `mcp.json.bak.2025-01-15T10-30-00-000Z`). New files are created without a backup.

**Idempotency** — If the `stacktape` entry already exists with identical configuration, the file is reported as `unchanged` and no backup is created.

## Examples

Install to all detected clients without interactive prompts:

```bash
stacktape mcp:add --autoConfirmOperation
```

Install in agent mode (JSONL output, no prompts):

```bash
stacktape mcp:add --agent
```
