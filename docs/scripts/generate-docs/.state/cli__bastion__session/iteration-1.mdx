# bastion:session

The `bastion:session` command opens an interactive shell on a deployed [bastion host](/resources/security/bastion-host). Use it to inspect private resources, run ad-hoc commands, or troubleshoot networking inside your VPC. The session runs over AWS Systems Manager (SSM) — no SSH keys, no open inbound ports.

## Usage

```bash
stacktape bastion:session --stage production --region eu-west-1
```

Your stack must contain at least one `bastion` resource. If the stack has a single bastion, Stacktape selects it automatically. If you have more than one, pass `--bastionResource` to pick which one.

```bash
stacktape bastion:session --stage production --region eu-west-1 --bastionResource myBastion
```

The command does not modify your stack and does not require a Stacktape configuration file — it reads the deployed stack state directly from AWS.

## How it works

When you run `bastion:session`, Stacktape:

1. Fetches the deployed stack's metadata to find the bastion's EC2 instance ID.
2. Calls AWS SSM `StartSession` targeting that instance.
3. Launches the AWS Session Manager plugin locally, connecting your terminal to the remote shell with full interactive I/O.
4. Terminates the SSM session when you exit the shell.

<Info>
The AWS Session Manager plugin must be installed on your machine. On most systems Stacktape bundles or downloads it automatically. If the session fails to start, verify the plugin is installed — see the [AWS Session Manager plugin install guide](https://docs.aws.amazon.com/systems-manager/latest/userguide/session-manager-working-with-install-plugin.html).
</Info>

## Important flags

| Flag | Description |
| --- | --- |
| `--stage` | **(required)** Stage of the deployed stack (e.g. `production`, `dev-jane`). |
| `--region` | **(required)** AWS region where the stack is deployed. |
| `--bastionResource` | Name of the bastion resource. Required only when the stack contains multiple bastions. |
| `--profile` | AWS profile to use for credentials. |

## Examples

### Connect to the only bastion in a stack

```bash
stacktape bastion:session --stage dev --region us-east-1
```

### Connect to a specific bastion by name

When your configuration defines more than one bastion, specify which one to connect to.

```bash
stacktape bastion:session --stage production --region eu-west-1 --bastionResource vpnBastion
```

### Use a specific AWS profile

```bash
stacktape bastion:session --stage staging --region eu-central-1 --profile my-aws-profile
```

## Errors

If the stack has no bastion resource, the command fails with a message indicating no resource of type `bastion` was found. Add a `bastion` resource to your configuration and redeploy before retrying.

If you pass a `--bastionResource` name that does not exist in the deployed stack, the command reports that the resource is not defined. Double-check the resource name matches your configuration.
