# preview-changes

The `preview-changes` command shows what would change in your deployed stack if you ran [`deploy`](/cli/deploy) with your current configuration. It packages workloads, compiles the CloudFormation template, validates it, requests a change set with property values, and displays a categorized summary of resource-level changes — without executing a deploy.

Use this command before deploying to verify that your configuration changes produce the intended infrastructure updates, catch unintended resource replacements, or review changes in a CI pipeline before approving a deployment.

## Usage

```bash
stacktape preview-changes --stage production --region eu-west-1
```

## How it works

The `preview-changes` command performs the same preparation steps as [`deploy`](/cli/deploy) — packaging workloads, resolving resources, compiling the CloudFormation template — but does not execute a stack update. Stacktape uploads the prepared CloudFormation template, validates it, and asks CloudFormation for a change set with property values included. The command then reads the proposed changes and displays a normalized summary grouped by Stacktape resource.

Changes are categorized into four types:

| Symbol | Label | Meaning |
|--------|-------|---------|
| `+` | new | A resource will be created |
| `-` | removed | A resource will be deleted |
| `!` | replaced | A resource will be destroyed and recreated (potential data loss) |
| `~` | updated | A resource will be modified in place |

Stacktape normalizes the raw CloudFormation diff to filter out internal runtime churn and dependency-only re-evaluations. If CloudFormation reports changes but none map to meaningful Stacktape resources, the output explicitly says so.

<Warning>
Resources marked as **replaced** will be destroyed and recreated. For stateful resources like databases or buckets, this can cause data loss. Review replacements carefully before deploying.
</Warning>

## Important flags

**`--preserveTempFiles`** — Keeps the compiled CloudFormation template and packaged artifacts in `.stacktape/[invocation-id]` so you can inspect them after the preview completes. Useful for debugging template-level differences.

**`--configPath`** — Points to a non-default configuration file. Use this when your `stacktape.ts` lives outside the current directory or has a non-standard name.

**`--agent`** — Enables agent-oriented output mode. The CLI wraps output in JSONL format and disables the interactive terminal UI. For `preview-changes`, the output includes a summary line with change counts (e.g. `SUMMARY: 1 new, 0 removed, 0 replaced, 2 updated`) followed by text lines per changed resource showing the action symbol, resource name, type, and highlights. Useful for CI pipelines or AI agent integrations that parse CLI output.

**`--outputFormat`** — Controls the CLI output format independently of `--agent`. Set to `jsonl` for machine-readable NDJSON, `plain` for uncolored text (auto-detected in CI environments), or `tty` for the full interactive UI. If omitted, the format is auto-detected. `--agent` implies `--outputFormat jsonl`.

## Examples

Preview changes for a staging environment:

```bash
stacktape preview-changes --stage staging --region us-east-1
```

Preview changes using a specific config file and preserve temporary files for inspection:

```bash
stacktape preview-changes --stage production --region eu-west-1 --configPath ./infra/stacktape.ts --preserveTempFiles
```

Preview changes in agent mode for CI pipeline consumption:

```bash
stacktape preview-changes --stage production --region eu-west-1 --agent
```

## FAQ

### Does preview-changes modify my deployed stack?

The command is initialized as non-modifying, but it still performs preparation work: it runs secret and SSM parameter preflights, packages workloads, uploads the compiled CloudFormation template, and validates it before requesting a change set. Your deployed infrastructure remains unchanged — no stack update is executed.

### How long does preview-changes take?

Duration depends on packaging work, template preparation, artifact upload, template validation, and the CloudFormation change-set request. Projects with many container images or large build steps take longer because workloads must be fully packaged before the change set can be requested.

### Can I use preview-changes in a CI pipeline?

Yes. Pass `--agent` to enable agent-oriented output mode. The CLI wraps output in JSONL format and disables interactive UI. The preview output includes a summary line with change counts and individual text lines per changed resource with action symbols (`+`, `-`, `!`, `~`), resource names, types, and highlights. Scripts can scan the output for the summary counts or the `!` replacement marker to gate deployments.

### What does "no meaningful Stacktape resource changes" mean?

CloudFormation sometimes reports internal dependency re-evaluations or runtime metadata shifts that don't correspond to actual infrastructure changes. Stacktape normalizes the raw diff and filters these out. When you see this message, the underlying stack would not change in any user-visible way.

### Does preview-changes require a deployed stack?

Yes. The command compares your current configuration against an already-deployed stack. `preview-changes` requires a deployed stack, so use [`deploy`](/cli/deploy) for the first deployment.

### How is preview-changes different from compile-template?

[`compile-template`](/cli/compile-template) exports the raw CloudFormation template to a file. It does not compare against a deployed stack or show what would change. `preview-changes` goes further: it packages workloads, uploads the template, validates it, requests a change set against the live stack, and shows a categorized diff of resource-level changes.

### What does a "replaced" resource mean?

A replaced resource will be destroyed and recreated by CloudFormation. This happens when you change an immutable property (like a database engine type or a Lambda function name). For stateful resources — databases, buckets, file systems — replacement means data loss unless you have backups. Always review replacements before deploying.

### Can I preview changes for a specific resource only?

No. The command previews the entire stack diff. CloudFormation change sets operate at the stack level and cannot be scoped to a single resource. The output groups changes by Stacktape resource name, so you can scan for the resource you care about.
