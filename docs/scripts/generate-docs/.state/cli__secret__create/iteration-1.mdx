# secret:create

The `secret:create` command stores a secret in AWS Secrets Manager in a specified region. Once created, the secret can be referenced anywhere in your Stacktape configuration using the [`$Secret('secret-name')`](/configuration/directives) directive — for example, as a database password, an API key, or a third-party token.

## Usage

```bash
stacktape secret:create --region eu-west-1
```

This launches an interactive wizard that prompts for a secret name and value. You can type the value directly or load it from a file.

### Non-interactive usage

When running in CI pipelines or from AI coding agents, pass all values as flags. The `--secretName` flag and either `--secretValue` or `--secretFile` are required in agent mode (`--agent`).

Create a secret with an inline value:

```bash
stacktape secret:create --region eu-west-1 --agent --secretName MY_API_KEY --secretValue sk-abc123
```

Create a secret from a file (the file contents are stored as JSON):

```bash
stacktape secret:create --region eu-west-1 --agent --secretName SERVICE_ACCOUNT --secretFile ./credentials.json
```

## Important flags

| Flag | Purpose |
|---|---|
| `--region` | **(required)** AWS region where the secret is stored. |
| `--secretName` | Name of the secret in AWS Secrets Manager. Required in agent mode; prompted interactively otherwise. |
| `--secretValue` | The plaintext value to store. Mutually exclusive with `--secretFile`. |
| `--secretFile` | Path to a file whose contents are stored as the secret value (serialized as JSON). |
| `--forceUpdate` | If a secret with the same name already exists, overwrite it without prompting. Only relevant in agent mode — interactive mode prompts for confirmation instead. |
| `--agent` | Disables interactive prompts and uses strict JSONL output. Requires `--secretName` and either `--secretValue` or `--secretFile`. |

## Updating an existing secret

If a secret with the given name already exists in the same region, the behavior depends on the mode:

- **Interactive mode** — Stacktape asks whether you want to update the existing secret's value. Choosing "no" aborts without changes.
- **Agent mode** — the command fails with an error unless you pass `--forceUpdate`. With `--forceUpdate`, the existing secret is overwritten silently.

```bash
stacktape secret:create --region eu-west-1 --agent --secretName MY_API_KEY --secretValue sk-new-value --forceUpdate
```

## Using secrets in your configuration

After creating a secret, reference it in any Stacktape configuration value with the `$Secret` directive. The directive resolves the secret value at deploy time from Secrets Manager.

## Examples

### Store a database password

```bash
stacktape secret:create --region us-east-1 --agent --secretName DB_PASSWORD --secretValue "p@ssw0rd-here"
```

### Store a multi-line service account JSON from a file

```bash
stacktape secret:create --region eu-west-1 --agent --secretName GCP_SERVICE_ACCOUNT --secretFile ./gcp-sa.json
```

### Rotate a secret in CI

```bash
stacktape secret:create --region eu-west-1 --agent --secretName DEPLOY_TOKEN --secretValue "$NEW_TOKEN" --forceUpdate
```

<Info>

Updating a secret's value does not automatically redeploy your stack. Resources that reference the secret via `$Secret()` pick up the new value on the next [deploy](/cli/deploy).

</Info>
