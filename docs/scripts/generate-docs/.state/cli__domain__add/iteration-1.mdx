# domain:add

The `domain:add` command registers a domain in your AWS account so Stacktape can provision TLS certificates and DNS records for it. Once added, the domain and its subdomains can be attached to [web services](/resources/compute/web-service), [static hosting](/resources/frontend/static-hosting), [HTTP API Gateways](/resources/networking/http-api-gateway), and other resources that accept [custom domains](/resources/networking/custom-domains).

## Usage

```bash
stacktape domain:add --region eu-west-1
```

The command runs an interactive wizard that guides you through domain verification and certificate provisioning. You must supply only the `--region` flag — the domain name itself is entered interactively.

## What it does

The `domain:add` wizard walks through up to four stages, stopping at the first incomplete step and resuming where it left off on subsequent runs:

Each stage stores its progress in AWS Parameter Store, so you can re-run the command later to continue from where you left off without repeating completed steps.

<Warning>Only root domains are accepted. If you enter a subdomain like `api.mydomain.com`, the command fails with an error. Add the root domain (`mydomain.com`) — subdomains are covered automatically by the wildcard certificate.</Warning>

## DNS ownership scenarios

The command handles two scenarios for DNS:

**Domain registered in Route 53.** If the domain was purchased through AWS Route 53, ownership is already verified and the command skips straight to TLS certificate provisioning.

**Domain registered elsewhere (GoDaddy, Namecheap, Hostinger, etc.).** The command creates a Route 53 hosted zone and prints the name server records you need to set at your registrar. DNS propagation can take hours to days. After propagation completes, re-run `domain:add` to continue with certificate provisioning.

<Tip>You can also manage domain verification status in the Stacktape Console at https://console.stacktape.com/domains. The Console shows real-time certificate validation state and alerts you when DNS propagation is complete.</Tip>

## TLS certificates

The command requests two free AWS ACM certificates:

- A **regional certificate** in the specified `--region` — used by Application Load Balancers and API Gateways.
- A **us-east-1 certificate** (if region is not already us-east-1) — required by CloudFront distributions (CDN).

Both certificates cover `mydomain.com` and `*.mydomain.com`. Validation is DNS-based — the command automatically adds the required CNAME records to the Route 53 hosted zone. Certificate issuance typically takes a few minutes after DNS records are in place.

## SES email verification

If domain ownership and TLS certificates are complete, the command optionally offers to verify the domain for sending email through AWS SES. This adds DKIM authentication records to the hosted zone. Skip this step if you don't plan to send transactional email from your application.

## Examples

Add a domain in the eu-west-1 region:

```bash
stacktape domain:add --region eu-west-1
```

Add a domain using a specific AWS profile:

```bash
stacktape domain:add --region us-east-1 --profile production
```

Add a domain using a named AWS account connected in the Console:

```bash
stacktape domain:add --region eu-west-1 --awsAccount my-prod-account
```

## FAQ

### Can I add a subdomain instead of a root domain?

No. The `domain:add` command only accepts root domains (e.g. `mydomain.com`). Subdomains like `api.mydomain.com` are covered automatically by the wildcard TLS certificate provisioned for the root domain. Configure subdomain routing in your resource's [custom domain settings](/resources/networking/custom-domains).

### Do I need to run this command in every region?

You need to run `domain:add` in each region where you plan to deploy stacks that use the domain. The command provisions a regional TLS certificate plus a us-east-1 certificate for CloudFront. If you only deploy in one region, one run is sufficient.

### How long does DNS propagation take?

DNS propagation after updating name servers at your registrar typically takes 1–48 hours, depending on the registrar and prior TTL values. AWS Certificate Manager validation usually completes within minutes once DNS is propagated.

### Is there a cost for adding a domain?

The command itself is free. AWS ACM certificates are free. A Route 53 hosted zone costs $0.50/month. If you registered the domain through Route 53, annual registration fees apply (starting at $3/year for `.click` domains).

### What happens if I cancel midway through the wizard?

Progress is saved to AWS Parameter Store after each completed step. Running `domain:add` again resumes from the last successful stage — you won't repeat certificate requests or hosted zone creation.

### Can I use a domain registered outside AWS?

Yes. The command creates a Route 53 hosted zone and prints the name servers you need to configure at your external registrar (GoDaddy, Namecheap, Hostinger, etc.). After propagation, re-run the command to continue.

### What resources can use a custom domain after adding it?

Once added, the domain can be attached to [web services](/resources/compute/web-service), [private services](/resources/compute/private-service) (via ALB), [HTTP API Gateways](/resources/networking/http-api-gateway), [static hosting buckets](/resources/frontend/static-hosting), and [SSR frontends](/resources/frontend/nextjs) (via CloudFront). Configure the domain in each resource's `customDomains` property.

### How do I verify the domain is ready?

After running the command, check certificate status in the Stacktape Console at https://console.stacktape.com/domains. Both the regional and us-east-1 certificates must show "ISSUED" status before the domain can be used in deployments.
