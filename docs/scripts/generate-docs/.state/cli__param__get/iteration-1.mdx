# param:get

Retrieve the runtime value of a [referenceable parameter](/configuration/referenceable-parameters) from a resource in a deployed stack. Use this command to get URLs, connection strings, ARNs, and other outputs without opening the AWS Console or parsing CloudFormation outputs manually.

## Usage

```bash
stacktape param:get --region eu-west-1 --resourceName myApi --paramName url
```

The command queries the deployed stack, locates the named resource, and prints the parameter's current value. It does not require a local Stacktape configuration file — only the region, resource name, and parameter name.

## When to use

Use `param:get` to pull a single runtime value from your deployed infrastructure. Common scenarios:

- Feed an endpoint URL into a local script or test harness.
- Retrieve a database connection string for a one-off migration tool.
- Read a resource ARN to pass into another CLI command or CI step.
- Verify the actual deployed value of a parameter after a deploy completes.

If you need a broader overview of all stack outputs and resources, use [`info:stack`](/cli/info-stack) instead.

## Examples

Retrieve the URL of a deployed web service named `api`:

```bash
stacktape param:get --region us-east-1 --resourceName api --paramName url
```

Retrieve a database connection string:

```bash
stacktape param:get --region eu-west-1 --resourceName mainDatabase --paramName connectionString
```

Use the stage flag when multiple stages share the same project name:

```bash
stacktape param:get --region eu-west-1 --stage production --resourceName cdn --paramName distributionDomainName
```

Capture the output in a shell variable for scripting:

```bash
API_URL=$(stacktape param:get --region eu-west-1 --resourceName api --paramName url)
```

## Error handling

If the resource name does not exist in the deployed stack, the command fails with an error identifying the stack and resource. If the parameter name is not valid for the given resource type, the error message lists all available referenceable parameters for that resource — making it easy to discover the correct name without checking documentation.
