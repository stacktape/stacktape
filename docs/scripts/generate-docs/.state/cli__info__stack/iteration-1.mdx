# info:stack

The `info:stack` command displays detailed information about a deployed stack, including its outputs (URLs, endpoints, resource identifiers) and the full list of AWS resources it contains. Use it after [deploying](/cli/deploy) to discover endpoints, verify resource creation, or feed stack data into scripts and CI pipelines.

## Usage

```bash
stacktape info:stack --region eu-west-1 --projectName my-project --stage prod
```

Alternatively, pass the stack name directly.

```bash
stacktape info:stack --region eu-west-1 --stackName my-project-prod
```

The `--region` flag is the only required argument. You identify the target stack in one of two ways: provide `--stackName` directly, or provide both `--projectName` and `--stage` (Stacktape derives the stack name as `projectName-stage`). If neither combination is supplied, the command exits with an error.

## Important flags

**`--stackName`** — The CloudFormation stack name, formatted as `projectName-stage`. Use this when you already know the full stack name. Mutually exclusive with the `--projectName` / `--stage` pair in practice — if `--stackName` is provided, the other two are ignored.

**`--projectName` and `--stage`** — An alternative to `--stackName`. Stacktape combines them into `projectName-stage` to locate the stack. Both must be provided together; supplying only one without the other (and without `--stackName`) causes an error.

**`--awsAccount`** — Name of the AWS account to query, as configured in the [Stacktape Console](/stacktape-console/connecting-your-aws-account). Required only when your organization has multiple connected AWS accounts and you need to target a specific one.

**`--agent`** — Switches output to strict JSONL for programmatic or LLM consumption. In agent mode the command prints the full stack details as a single JSON object instead of the interactive table view. Useful in CI scripts or when piping output to `jq`.

**`--outputFormat`** — Controls the output format independently of `--agent`. Accepts `jsonl`, `plain`, or `tty`. When omitted, the format is auto-detected from the environment (`tty` in an interactive terminal, `plain` in CI).

## Examples

Inspect a production stack and get its outputs (URLs, identifiers).

```bash
stacktape info:stack --region us-east-1 --projectName my-api --stage prod
```

Use the combined stack name instead of separate project and stage flags.

```bash
stacktape info:stack --region us-east-1 --stackName my-api-prod
```

Get machine-readable JSON output for a CI pipeline or scripting.

```bash
stacktape info:stack --region eu-west-1 --projectName my-api --stage staging --agent
```

Query a stack in a specific connected AWS account.

```bash
stacktape info:stack --region us-west-2 --projectName payments --stage prod --awsAccount production-account
```

## Related commands

- [`info:stacks`](/cli/info-stacks) — List all stacks deployed in a region. Use this to discover stack names before drilling into a specific one with `info:stack`.
- [`info:operations`](/cli/info-operations) — View recent deployment operations and their success/failure status for a project or stage.
- [`deploy`](/cli/deploy) — Deploy or update a stack. After deploying, use `info:stack` to retrieve the resulting outputs and endpoints.
- [`param:get`](/cli/param-get) — Retrieve a single parameter value from a resource in a deployed stack, rather than the full stack overview.
