# debug:alarms

The `debug:alarms` command displays the current state of all CloudWatch alarms associated with a deployed stack. Use it to quickly check which alarms are firing, which are healthy, and which lack sufficient data — without opening the AWS Console.

## Usage

```bash
stacktape debug:alarms --stage production --region eu-west-1
```

## What it does

Stacktape queries CloudWatch for all alarms prefixed with the stack name and displays their current state. Each alarm shows:

- **State** — `OK`, `ALARM`, or `INSUFFICIENT_DATA`
- **Resource** — the Stacktape resource the alarm belongs to (when identifiable)
- **Metric** — the CloudWatch metric being monitored (e.g. `Errors`, `CPUUtilization`)
- **Threshold** — the numeric threshold that triggers the alarm
- **Comparison** — how the metric is compared to the threshold (e.g. `GreaterThanThreshold`)
- **Last updated** — when the alarm state last changed
- **Reason** — why the alarm is firing (shown only for alarms in `ALARM` state)

The command is read-only and does not modify any resources.

## Important flags

**`--resourceName`** filters results to alarms belonging to a single Stacktape resource. Useful when your stack has many resources and you want to inspect one service at a time.

```bash
stacktape debug:alarms --stage production --region eu-west-1 --resourceName apiFunction
```

**`--state`** filters alarms by their current state. Accepted values are `OK`, `ALARM`, and `INSUFFICIENT_DATA`. Use this to find only the alarms that need attention.

```bash
stacktape debug:alarms --stage production --region eu-west-1 --state ALARM
```

**`--agent`** switches output to JSON format (NDJSON), which is useful for programmatic consumption by scripts or AI coding agents.

```bash
stacktape debug:alarms --stage production --region eu-west-1 --agent
```

## Examples

List all alarms for a staging stack:

```bash
stacktape debug:alarms --stage staging --region us-east-1
```

Show only alarms that are currently firing:

```bash
stacktape debug:alarms --stage production --region eu-west-1 --state ALARM
```

Check alarms for a specific Lambda function:

```bash
stacktape debug:alarms --stage production --region eu-west-1 --resourceName paymentHandler
```

Combine both filters to check if a specific resource has any active alarms:

```bash
stacktape debug:alarms --stage production --region eu-west-1 --resourceName paymentHandler --state ALARM
```
