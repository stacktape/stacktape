# defaults:configure

The `defaults:configure` command sets system-wide default values for common Stacktape CLI arguments. Once configured, these defaults apply to every command that accepts them, so you don't need to pass `--region`, `--stage`, or `--projectName` on every invocation.

## Usage

```bash
stacktape defaults:configure
```

The command runs interactively. It prompts you for each configurable property one at a time, showing the current value (or `<< not-set >>` if unset). For each prompt:

- **Leave blank** to keep the current value unchanged.
- **Enter a new value** to update it.
- **Enter a space** to unset (clear) the value back to its default.

Sensitive values like the API key are masked in the prompt output.

## Configurable defaults

The command walks through two groups of settings.

### CLI argument defaults

These defaults are applied automatically when the corresponding CLI flag is not explicitly provided.

| Property | Description | Default if unset |
|---|---|---|
| `region` | AWS region | None |
| `profile` | AWS profile | `default` |
| `stage` | Stage | None |
| `projectName` | Project name | None |
| `awsAccount` | AWS Account name | None |

Setting a default `region` and `stage` is the most common use case — it saves you from typing `--region eu-west-1 --stage production` on every deploy, delete, or debug command.

### Other defaults

| Property | Description | Default if unset |
|---|---|---|
| `apiKey` | Stacktape API Key | None (sensitive — masked in output) |
| `executablePython` | Python executable path | `python` |
| `executableJavascript` | Node.js executable path | `node` |

The executable settings control which runtime binary Stacktape uses when resolving directives that invoke Node.js or Python scripts. Most users never need to change these unless they use a non-standard installation path or version manager.

## Where defaults are stored

Defaults are persisted to `~/.stacktape/persisted-state.json`. This file is local to your machine and is not checked into version control. The command prints the full path after saving.

<Info>
The `defaults:configure` command does not require an API key. You can run it before logging in to pre-configure your region and stage.
</Info>

## Examples

Set your default region and stage so most commands work without extra flags.

```bash
stacktape defaults:configure
```

After running the command and entering `eu-west-1` for region and `production` for stage, subsequent commands like `stacktape deploy` will use those values unless you override them with explicit flags.

To check what defaults are currently configured, use [`defaults:list`](/cli/defaults-list).

```bash
stacktape defaults:list
```

Explicit CLI flags always override configured defaults. For example, even if your default region is `eu-west-1`, running `stacktape deploy --region us-east-1` deploys to `us-east-1`.

## FAQ

### Can I set defaults per project instead of globally?

No. The `defaults:configure` command sets machine-wide defaults stored in `~/.stacktape/persisted-state.json`. They apply to all projects on your machine. If you work on multiple projects with different regions or stages, pass the flags explicitly or use shell aliases for project-specific workflows.

### Do explicit CLI flags override defaults?

Yes. Any flag you pass on the command line takes precedence over the configured default. For example, `--region us-east-1` overrides a default region of `eu-west-1` for that single invocation.

### How do I clear a configured default?

Run `stacktape defaults:configure` again. When prompted for the property you want to clear, enter a single space character. This resets it to the built-in default (or unset, if there is no built-in default).

### Is my API key stored securely?

The API key is stored in `~/.stacktape/persisted-state.json` as plain text on your local filesystem. The file is not encrypted, so protect it with standard filesystem permissions. The `defaults:configure` prompt masks the key when displaying the current value, but the stored value is not encrypted at rest.

### Do I need to be logged in to run defaults:configure?

No. The `defaults:configure` command does not require a Stacktape API key. You can configure region, stage, and profile defaults before logging in.
