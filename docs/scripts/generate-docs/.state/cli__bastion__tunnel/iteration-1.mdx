# bastion:tunnel

The `bastion:tunnel` command creates a secure SSH tunnel through a [bastion host](/resources/security/bastion-host) to reach resources inside a private VPC. Use it to connect local tools — database GUIs, Redis CLIs, or HTTP clients — to resources that have no public endpoint. The tunnel uses AWS Systems Manager (SSM) port forwarding, so no SSH keys or open inbound ports are needed.

## Usage

```bash
stacktape bastion:tunnel --stage prod --region eu-west-1 --resourceName myDatabase
```

The command opens one or more local ports, prints the local-to-remote mapping, and keeps running until you press `Ctrl+C`. Your stack must contain a [bastion](/resources/security/bastion-host) resource for this to work.

## Supported resource types

Bastion tunneling works with four resource types:

| Resource type | What gets tunneled |
|---|---|
| [relational-database](/resources/databases/relational-database) | Primary endpoint. Reader endpoint and read replicas if present. |
| [redis-cluster](/resources/databases/redis) | Primary endpoint. Reader endpoint if present. |
| [application-load-balancer](/resources/networking/application-load-balancer) | Every exposed listener port. |
| [private-service](/resources/compute/private-service) | Every exposed listener port (requires the private service to use an application load balancer). |

If you target a resource type not listed above, Stacktape exits with an error explaining which types are supported.

## Important flags

**`--resourceName`** (required) — the name of the target resource you want to reach through the tunnel, as defined in your Stacktape configuration.

**`--bastionResource`** — the name of the bastion resource to tunnel through. Only needed when your stack contains more than one bastion. If omitted and there is exactly one bastion, Stacktape selects it automatically.

**`--localTunnelingPort`** — the local port to bind the tunnel to. If not specified, Stacktape picks an available port automatically. When tunneling to a resource with multiple endpoints (e.g. a database with a reader replica), subsequent endpoints use the next available ports starting from this number.

## Examples

### Tunnel to a relational database

Connect a local database GUI (e.g. pgAdmin, DataGrip) to a VPC-only PostgreSQL or MySQL database.

```bash
stacktape bastion:tunnel --stage prod --region eu-west-1 --resourceName myDatabase
```

The output shows the local address to use in your client, along with the substituted connection string.

### Tunnel on a specific local port

Pin the local port so bookmarked database connections stay stable.

```bash
stacktape bastion:tunnel --stage prod --region eu-west-1 --resourceName myDatabase --localTunnelingPort 5433
```

### Tunnel to a Redis cluster

Access a VPC-only Redis cluster from a local `redis-cli` or Redis GUI.

```bash
stacktape bastion:tunnel --stage prod --region eu-west-1 --resourceName myRedis
```

### Tunnel to a private service

Reach a [private service](/resources/compute/private-service) that sits behind an application load balancer. The tunnel opens a local port for each exposed listener port.

```bash
stacktape bastion:tunnel --stage prod --region eu-west-1 --resourceName myPrivateApi
```

### Specify which bastion to use

When a stack has multiple bastion hosts, pass the bastion name explicitly.

```bash
stacktape bastion:tunnel --stage prod --region eu-west-1 --resourceName myDatabase --bastionResource devBastion
```

## How tunneling works

1. Stacktape looks up the target resource in the deployed stack and resolves its private host and port.
2. It identifies the bastion host's EC2 instance ID (either the only bastion in the stack or the one specified with `--bastionResource`).
3. An SSM port-forwarding session is started from your machine to the bastion instance, forwarding traffic to the target's private endpoint.
4. The command prints the local address (`127.0.0.1:<port>`) mapped to each remote endpoint and keeps the tunnel open.
5. When you press `Ctrl+C`, all tunnel sessions are closed.

For databases with read replicas or reader endpoints, Stacktape opens additional tunnels automatically — one per endpoint — and prints all mappings.

<Info>
The tunnel uses SSM Session Manager under the hood, which means it works without opening any inbound ports on the bastion. The [AWS SSM Session Manager plugin](https://docs.aws.amazon.com/systems-manager/latest/userguide/session-manager-working-with-install-plugin.html) must be installed on your machine.
</Info>
