# issues:resolve

The `issues:resolve` command marks a runtime issue as resolved. Stacktape automatically reopens the issue if the same error occurs again in your deployed functions or containers, so resolving an issue signals that you believe the underlying cause has been fixed.

## Usage

```bash
stacktape issues:resolve --issueId iss_abc123def456
```

You need the issue ID, which you can find by running [`issues:list`](/cli/issues-list). Once resolved, the issue moves out of the active issues view. If the same error recurs after resolution, Stacktape reopens the issue automatically — no manual intervention needed.

## When to use

Resolve an issue after you have deployed a fix for the underlying error. This keeps your issues list focused on unresolved problems and lets Stacktape's automatic reopening act as a regression detector. If you want to permanently suppress an issue regardless of future occurrences, use [`issues:ignore`](/cli/issues-ignore) instead.

## Important flags

| Flag | Required | Description |
|------|----------|-------------|
| `--issueId` (`-iid`) | Yes | The ID of the issue to resolve |
| `--agent` (`-ag`) | No | Enables machine-readable JSONL output for programmatic use |
| `--outputFormat` (`-ofmt`) | No | Controls output format: `jsonl`, `plain`, or `tty` |
| `--logLevel` (`-ll`) | No | Log verbosity: `info`, `debug`, or `error` |

## Examples

Resolve an issue by its ID:

```bash
stacktape issues:resolve --issueId iss_abc123def456
```

Resolve an issue in agent mode (useful in scripts or when called by AI coding assistants):

```bash
stacktape issues:resolve --issueId iss_abc123def456 --agent
```
